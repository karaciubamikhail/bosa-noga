# Интернет-магазин обуви (SPA)

Разработка клиентского приложения интернет-магазина обуви с полноценной пользовательской логикой: каталог, поиск, карточка товара, корзина и оформление заказа.  
Проект реализован на основе готовой вёрстки и внешнего API без возможности изменения серверной части.

---

## Решённые задачи

- Реализовано SPA-приложение интернет-магазина обуви
- Настроена постраничная маршрутизация
- Реализована работа с HTTP-запросами
- Добавлено глобальное состояние корзины
- Реализовано оформление заказа
- Добавлены загрузчики и обработка ошибок
- Внешний вид полностью соответствует предоставленной HTML-разметке

Результат работы представлен в данном Git-репозитории.

---

## Сущности

**Товар**  
Карточка товара с описанием, изображениями, ценой, доступными размерами и наличием.

**Категория**  
Тип обуви, используемый для фильтрации товаров.

**Размер**  
Размер обуви. Может быть доступен или недоступен для покупки.

**Корзина**  
Локальное хранилище выбранных товаров.  
Одна позиция = товар + размер.

**Заказ**  
Данные пользователя (телефон, адрес) и список товаров, отправляемые на сервер.

## Работа с API

### Основные эндпоинты

- Хиты продаж  
  `GET /api/top-sales`

- Категории  
  `GET /api/categories`

- Каталог товаров  
  `GET /api/items`  
  `GET /api/items?categoryId=X`  
  `GET /api/items?offset=6`

- Поиск  
  `GET /api/items?q=searchText`

- Карточка товара  
  `GET /api/items/:id`

- Оформление заказа  
  `POST /api/order`

---

## Корзина

- Данные корзины хранятся в `localStorage`
- Стоимость товара фиксируется в момент добавления
- Один товар + размер = одна позиция
- Общая стоимость рассчитывается автоматически
- После успешного оформления заказа корзина полностью очищается

---

## Реализация проекта

- Проект реализован в виде SPA
- Сервер используется как внешний API
- Серверная часть не модифицируется
- Интерфейс полностью соответствует предоставленной вёрстке
